How I Work

The work is public and social. The best collaborations come from shared context — seeing how I think, watching how the tools behave, playing at the same table.

RPG Wednesday

A recurring Wednesday tabletop session. Watching how someone runs a table or plays a character tells you something real about how they think, collaborate under uncertainty, and treat other people's ideas.

If you play tabletop RPGs and want to see how I work with a group, this is the most honest context available.

Pretext.js

A measurement-driven text layout library. No DOM reflow. Reads body copy, writes CSS custom properties, lets layouts breathe correctly at any size. The code is the calling card.

If you're an engineer evaluating the work, read the source. The quality of the library tells you more than a portfolio page.

spw-workbench

The public parser, runtime, and tooling work. Where the grammar research and systems thinking are most visible. Where I push on deep algorithmic problems.

Long-horizon work, visible in public. The kind of project that takes years and produces durable things.

This site

The site itself is a working system: semantic HTML as a conceptual ledger, CSS as material physics, JS as readable state transitions. Every component has a contract. The interaction model is intentional.

Inspect the source. The architecture is the portfolio.

Sometimes I have bills. Sometimes I need supplies, food, or time. This card is for making those asks feel like what they are — honest requests from a person doing real work — rather than appeals to guilt or charity. The aesthetic matters because dignity matters.

Configure the ask, screenshot it, share it. The same infrastructure as the services card — because a personal need and a professional offer are both honest transactions.
