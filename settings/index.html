~"settings_notes"

Notes

Why quiet by default?

The map and console are inspection tools that should not compete with the page content.

What do layer dispositions do?

They change how a card announces itself: orient opens, attune sharpens, compose gathers, project broadcasts, and settle lets the layer rest.

Conceptual vs realized?

Conceptual surfaces frame or compare an idea; realized surfaces present an artifact or committed output. Substrate names the operator family the surface is leaning on, and phrasing tells you whether it is acting as premise, guide, inquiry, structure, reflection, instruction, or artifact.

What activates frames?

Hash links, clicks, focus, hover, map selection, console controls, and mode switches. The viewport setting only controls scroll-driven activation.

Hidden chrome?

Return to /settings/ from the header or direct URL and choose quiet or full display again.

^"runtime_map"{

Runtime Map

Settings do not only change visible controls. They update stored values, derived architectural modifiers, document datasets, and CSS custom properties that other surfaces read.

1. Stored Settings

Local values are normalized against the site settings registry before they are applied.

DEFAULT_SITE_SETTINGS → normalizeSiteSettings()

2. Derived Modifiers

Settings become modifiers for typography, motion, grain, climate, ecology, and semantic behavior.

deriveArchitecturalModifiers(settings)

3. Document Datasets

The runtime writes stable data-spw-* attributes to the root for CSS and JS to read.

html[data-spw-color-mode] · html[data-spw-semantic-density]

4. CSS Variables

Typography, motion, grain, and other values are expressed as custom properties for route surfaces.

--site-root-font-size · --site-line-height · --grain-opacity

5. Surface Readers

Shell, components, route surfaces, and optional JS modules read those values to decide behavior.

shell → components → routes → enhancements
