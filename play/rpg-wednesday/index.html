Play Surface · Recurring Session

RPG Wednesday

A recurring Wednesday tabletop RPG session. This surface holds the session log, world, and cast — each session is a dated event; the world and characters are the persistent frame beneath them. The local gameplay kit below stores private table state in this browser for use during play.

The next design layer is a local media staging table: players upload their own character, item, place, and artifact images, frame them in screenshot-ready cards, then manually capture those cards for external image or animation tools.

#>rpg_wednesday
#:layer #!play
#:surface #!event_log

^"structure"{
 sessions: ~"./sessions/"   @event  session log, dated /YYYY-MM-DD/
 world:    ~"./world/"      ^object  stable setting and lore
 cast:     ~"./cast/"       ~refs    character reference nodes
 arcs:     ~"./arcs/"       ?[probe  narrative thread groupings
}

Sessions are events at a specific moment in time. The world is a stable object that sessions act inside. Cast members are reference nodes — they point outward to the sessions where they appeared. Arcs are probes: open questions carried across multiple sessions.

The URL structure here was derived directly from the Spw operator model. See the grammar surface and the deeper spw-workbench notes.

play surface rpg wednesday @ sessions ^ world ~ cast ?[ arcs @ local kit @ capture frames
@local_gameplay_kit

Local Gameplay Kit

Enable JavaScript to use private localStorage tools for scene state, initiative, clocks, notes, a table dashboard, and copyable session-log seeds. Static campaign routes remain available either way.

@asset_capture_frames

Asset Capture Frames

Planned next: a private, browser-local asset register for images players provide themselves. The site will stage the image and text; the player takes the screenshot and chooses what to do with it outside the site.

4:5 Character portrait Name, role, faction, palette, and mood in a clean vertical frame.
1:1 Item card One subject, strong silhouette, optional tags, and a concise prompt band.
16:9 Place card Location, scene context, weather, threat level, and wide screenshot composition.
9:16 Threat reveal A mobile-first vertical frame for foes, omens, clocks, and dramatic reveals.

Storage boundary

Images should live in IndexedDB; localStorage should keep only metadata such as ids, labels, tags, selected motif, and active capture preset.

Prompt companion

Prompt text should describe subject, role, mood, palette, scene context, style constraints, and negative notes without hard-coding any external service workflow.

Screenshot mode

Controls recede, contrast increases, the frame locks to an aspect ratio, and an exit path remains visible. No automatic capture is required.

Exploration motifs

Operator palettes, land-cluster tones, constellation marks, archive stamps, and brace treatments can become optional local presentation rewards.

Surface Register
