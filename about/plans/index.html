Strategic Landscape

Plans & Horizons

This page tracks active and future plans for spwashi.com. Each plan is not just a topic; it is a structured action surface. The point is to make the plans easier to read, easier to connect, and easier to visualize later as a semantic field rather than a pile of notes.

For now, the strongest improvement is to make the verbs clearer. A good plan should reveal what it is trying to refine, build, map, integrate, teach, stage, publish, or adapt. That gives the page stronger structure and prepares it for later interactive or visual tooling.

Verb Register

Plans become more useful when their verbs are explicit. These are the verbs currently structuring the work. Later, each one could become interactive, filterable, or visually projected as part of a plan map.

Refine

Clarify operator meaning, naming, behavior, and conceptual precision.

Build

Create surfaces, runtimes, tools, pages, and reusable structures.

Map

Expose relationships, symmetry, spatial address, discovery routes, and asset classification.

Integrate

Make SVG, runtime settings, images, gestures, and surfaces work coherently together.

Teach

Help visitors build fluency through explanation, repetition, and experience.

Stage

Control timing, pacing, cinematic reveal, and semantic emphasis over time.

Publish

Make plans, assets, operator pages, and media legible as public artifacts.

Adapt

Support mobile runtime, accessibility, settings, and user preference without losing coherence.

Operator Semantics & Lore

Plans that refine how operators work, what they mean, and how they become public surfaces rather than private theory.

Operator Semantics Refinement

foundation

Refine Spw operators so they remain true to the established lore and original set while leaving room for interpretation through symmetry, priming, and operator physics.

refine map teach
Feature Considerations:
  • Document operator lineage from original meaning to current implementation.
  • Explain physics: spatial acts, priming properties, interaction semantics.
  • Map symmetry: operator pairs, coherent dynamics, mirrored functions.
  • Clarify why some sequences feel inevitable once the physics is understood.
  • Show how interpretation emerges from structure rather than arbitrary rules.
Wonder Hook:

Operators are not arbitrary symbols. They are repeatable spatial and attentional relationships. Once that becomes visible, the language stops feeling invented and starts feeling discovered.

Spw Operator Pages

active

Create a public atlas where each operator gets a stable page before it gets richer runtime behavior. The page should teach interaction semantics, metaphysics, brace physics, and symmetry through its own structure.

build teach publish
Feature Considerations:
  • One stable page per operator.
  • Sections for interaction, metaphysics, brace physics, and symmetry.
  • Operator-aware styling so the page demonstrates the operator through the medium itself.
  • Public discovery from home, software, and route navigation.
  • Stable URLs that can be referenced from other pages and future diagrams.
Wonder Hook:

The medium should teach the message. A reference operator page should feel referential in how it stages explanation and movement, not just in what it says.

Plan Discovery & Use

What is a plan?

A plan is a structured document for a feature consideration or brand horizon. It should be usable as guidance, not just archival prose.

What do the statuses mean?

  • Foundation: conceptually strong and ready to guide implementation
  • Active: being developed or refined now
  • Planning: the direction is set, but not yet underway

What should improve later?

The next step is visual structure: better surface styling, verb-aware filtering, plan relationship diagrams, and a more legible semantic map.

Where are the source files?

Full plan documents live in the GitHub repository under `.agents/plans/...`, and related `.spw` files live in the same repo where applicable.
